# Simes 1.4.0

Simes（Integrated Mod Enhancement Suite）是由 **7imes** 为 SimMC 开发的 Minecraft 1.21.8 Fabric 客户端整合增强模组。

- 项目主页：https://github.com/Nov7imes/Simes
- 适用服务器：`play.simmc.cn`
- 市场数据来源：Simall
- 无需安装在服务端

## 安装

1. 安装 Minecraft 1.21.8、Fabric Loader 0.17.2 或更高版本、Java 21，以及适用于 1.21.8 的 Fabric API。
2. 删除旧的 Simes、SimPriceDisplay 和独立 AutoMessage JAR，避免重复加载。
3. 将 `Simes-1.4.0-mc1.21.8.jar` 放入客户端的 `.minecraft/mods/`。

## 市场价格与坐标

- 在物品栏、背包或容器中悬停物品，即可在提示框底部查看最低出售、最高收购、店铺、港口、库存、数据更新时间和 Simall 数据来源。
- 使用游戏内真实显示名称匹配物品，支持由原版物品加 NBT/组件制作的插件物品；优先精确匹配，唯一且可信的模糊结果也可使用。
- 默认悬停后按 `C` 复制最低出售店铺坐标，按 `V` 复制最高收购店铺坐标；可在“选项 → 控制”中改键，也支持鼠标键。
- 悬停物品后按一次 `Ctrl` 可锁定该物品并打开市场详情，显示最低出售和最高收购各前五条；锁定后可将鼠标移入面板选择任意报价，左键或对应的 `C`/`V` 快捷键复制坐标。
- 锁定期间鼠标经过其他物品不会自动切换；将鼠标放在新物品上再按 `Ctrl` 可直接切换，按 `Esc`、再次对同一物品按 `Ctrl` 或关闭容器会解锁。
- 详情锁定期间会隐藏原物品提示框和悬停价格，并冻结所选物品的逻辑状态；面板下方的背包点击会被拦截，避免移动鼠标选择报价时误拿、误放物品。
- `price_index.json` 每 10 分钟尝试更新，作为当前最佳价的最高优先级；`shops.json` 每 1 小时尝试更新，提供其他完整报价。面板分别显示最佳价和全量缓存的更新时间。
- 两类市场数据均支持 gzip、ETag 和 HTTP 304，彼此独立更新。任一下载、解析或校验失败时继续使用对应的上次有效缓存。
- 全量文件使用流式解析，客户端只为每件物品保留出售/收购各前五条，不将全部原始报价常驻内存。
- 功能只在 `play.simmc.cn` 启用，缓存位于 `.minecraft/config/simes/`。

## 背包与仓库价值

- 打开背包时，在 GUI 右下区域显示“背包价值”和“今日收益”。
- 物品提示框使用更高显示层级；即使与价值面板重叠，物品说明也不会被面板遮挡。
- 打开箱子、木桶及服务器双箱容量容器时，显示仓库价值、随身背包价值、总计、已估价与未估价格数。
- 价值按“物品数量 × 稳健收购估价”计算：先剔除库存不足 100 的收购报价，排除剩余报价中的第 1 名，再对第 2～5 名取中位数；不足两条有效报价或没有收购价的物品不计入。精确匹配和可靠模糊匹配都会参与估值。
- 盔甲、快捷栏和副手物品计入背包价值。
- “开关背包/仓库价值显示”快捷键可在游戏控制设置中修改，状态会保存。

### 多箱累计 `/cal`

- 输入 `/cal` 开始记录，再次输入 `/cal` 结束并在本地聊天输出箱子数量和总价值。
- 每关闭一个容器，显示其序号、X/Y/Z、该箱价值和当前累计。
- 使用维度与方块坐标去重；大箱左右两半视为同一容器，木桶使用自身坐标。
- 记录只保存在本次客户端会话中，不会上传。

## 余额与今日收益

- 登录目标服务器 3 分钟后首次自动查询 `/bal`，之后每 10 分钟更新。
- 只隐藏 Simes 自动查询产生的余额回复，玩家手动执行 `/bal` 时仍正常显示。
- 今日收益按“最新余额 − 当日本地基准余额”计算；同一天重复登录会沿用当天基准，日期变化后自动重建。
- 查询失败时保留最后一次有效余额，并在下一周期重试。

## 奥术冷却 HUD

- 输入 `/simes` 或 `/simes hud` 打开统一设置；“按键设置”页可直接点击功能并按下新的键盘键或鼠标键，按 `Esc` 清除绑定，无需跳转原版控制界面。
- 可选择保留原版 Action Bar，或使用 Simes 自定义奥术冷却 HUD。
- 自定义模式支持多个奥术同时显示、20 个奥术图标、法术专属加粗配色、平滑倒计时、进出场和补位动画。
- 同一次冷却按单调递减处理，过滤临近 0 秒时乱序到达的旧数据，避免数字回弹。
- 会识别并隐藏服务器盾牌产生的空白/红格 Action Bar；混合数据会先剥离盾牌部分，再读取奥术冷却。
- 支持通用吟唱 HUD：直接读取服务器 Boss Bar 的奥术名和实时进度，正常完成与中途打断均可识别；雷击、火陨术等不同时长和颜色无需单独适配。
- 支持持续状态 HUD：腾云、凌步、御风、寒冰吐息和火焰吐息等服务器 tick 状态统一显示为递减状态条。
- 吟唱、持续状态和公共冷却均使用对应奥术的图标与专属颜色。服务器没有连续推送公共冷却时，会在成功释放后使用内置表本地计时，并接受服务器提示校准；低于 1 秒不显示。
- Simes 模式只隐藏已识别的吟唱/持续状态 Boss Bar，普通 Boss 血条和其他服务器 Boss Bar 保持原样。
- 被 Simes 隐藏的 Boss Bar 会按 UUID 维护完整数据包生命周期；即使服务端延迟发送名称或进度更新，也不会将缺少 `ADD` 的更新交给原版 HUD，兼容 FCL/安卓端较不稳定的包处理时序。
- 在 HUD 设置中点击“HUD 布局与缩放”即可调整奥术 CD、吟唱/持续/公共冷却、自动消息和价值面板。背包价值与仓库价值是两个独立 HUD，各自保存位置和缩放；仓库预览中可分别选择并拖动二者。选择对应布局后会显示模拟背包或双箱界面作为定位参考，靠近屏幕中心或 GUI 边缘时会吸附并显示辅助对齐线。
- HUD 配置保存于 `.minecraft/config/simes/hud.json`。

## 自动消息

- 按默认 `O` 键或输入 `/atmsg` 打开设置；快捷键可在控制设置中修改。
- 设置消息/命令及 1–6400 秒发送间隔，然后点击“保存并返回”。以 `/` 开头的内容按命令发送，否则按聊天消息发送。
- 输入 `/atmsg st` 启动，再次输入则停止。
- 运行期间倒计时 HUD 始终显示，停止后隐藏，不占用 Action Bar。
- 启动后以及每次进入服务器时都会先完整执行一次设定的倒计时；到期后会在玩家和网络可用时发送一次并开始下一周期，不会刚进服立即发送。
- 旧 `.minecraft/config/automessage.json` 会迁移至 `.minecraft/config/simes/automessage.json`。

## 发酵与厨具辅助

- 可在 `/simes` 中分别开关“发酵辅助”和“厨具辅助”。
- 准星对准 10 格内的发酵桶、炖锅、蒸锅或煎锅时，会在目标方块上方显示半透明信息卡。
- 发酵桶会根据玩家物品栏的服务端同步结果记录投入与取出；单种食材本地记录上限为 10 个。
- 使用服务器的烹饪时钟可同步桶内材料和服务器权威剩余时间；同步后客户端以单调时钟平滑推算倒计时，归零只显示“预计已到”，必须再次使用烹饪时钟并收到服务器完成回复后才显示“已完成（服务器确认）”。
- 发酵被开桶等操作中断或重新开始后，旧时间立即失效，信息卡会提示重新使用烹饪时钟校准。
- 重新登录后，本地材料记录不会被当作服务器事实恢复；准星重新识别发酵桶时显示“状态未知”，使用烹饪时钟即可获取当前服务器状态。
- 普通厨具的开盖/合盖、材料和完成变化来自客户端可见实体；烹饪时间属于本地预估，不代表服务器权威结果。
- 功能灵感来自 Crazy_me 的 ChefHelper；Simes 中的实现为独立重新设计与编写。

## Debug 日志

- 输入 `/debug start` 开始记录，复现问题后输入 `/debug end` 结束；单次最多 10 分钟，达到上限会自动保存并结束。
- 记录服务端 Action Bar 数据包、系统提示、主副手物品及组件、状态效果、经验和界面/价值面板状态。
- 战斗诊断会记录攻击输入、手持武器、攻击冷却、准星目标，以及 96 格内生物的血量、吸收值、受伤、死亡和浮空文字变化。
- 同时监听 Boss Bar、实体战斗数据包、粒子、声音、爆炸、死亡与 Custom Payload，用于分析完全由服务端驱动的战斗系统。

## 诊断与兼容性

- 输入 `/simes diagnose` 会在 `.minecraft/config/simes/diagnostics/` 生成本地诊断 ZIP，包含运行环境、模组版本列表、Simes 配置及最近一次 Debug 日志的脱敏副本。
- 诊断内容会替换用户目录、UUID 和 IPv4 地址；最近日志最多收集 2 MiB。诊断包不会自动上传，发送前仍应由玩家自行确认内容。
- 启动时检查 Fabric API、Java 主版本以及旧版独立 AutoMessage/SimPriceDisplay；仅发现风险时在日志和登录提示中报告。
- `simes.json` 与 `hud.json` 带配置格式版本，旧配置会在读取后自动迁移和补全。
- 日志位于 `.minecraft/config/simes/debug/`，文件名精确到秒，仅保存在本地，不自动上传。

## 构建与公开包

需要 Java 21：

```text
gradlew.bat clean build
```

正式发布包使用 Fabric Loom 生成的标准 `remapJar`，不进行代码混淆，以保证 PCL、Fabric Loader 和其他模组的兼容性。
