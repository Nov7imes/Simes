- 自动消息倒计时改用仅存在于内存中的单调时钟；重新排定发送不再反复写入配置文件，只在设置或启用状态实际变化时持久化，减少无意义磁盘 I/O，并避免系统时间调整干扰倒计时。
- 自动消息的语言资源合并至 `assets/simes`，移除旧的独立资源命名空间与 JAR 内 `LICENSE_automessage`。
- 许可升级为可用于 7imes 多个项目的 `7imes Source-Available License 1.0`，不再限定于 Simes；历史版本已经授予的许可权利不受影响。
- 完成公开发布前资料治理：README、安装包名、运行环境、Debug 时限、AutoMessage 重连行为、发酵与厨具说明均与 1.4.0 实际实现对齐。
- 项目许可由 MIT 调整为源码可查看许可：允许个人使用官方未修改版本及查看源码，但未经作者书面许可不得二次分发、改作发布、商业利用或冒充作者。
- 正式 JAR 现在附带 README，GitHub 仓库新增 1.4.0 发行摘要和 Java 21/Fabric 自动构建测试工作流。
- 修复市场请求 `User-Agent` 仍固定报告 1.3.7 的历史遗留；现在自动读取 Fabric 元数据中的当前 Simes 版本，后续升级无需重复修改。
- 修正 Debug 启动日志仍标记 `max=60s` 的旧文字，实际与说明统一为单次最多 10 分钟。
- 扩展 `/debug` 的厨具与发酵诊断：逐槽记录玩家物品栏和当前 GUI `ScreenHandler` 的实际变化，并记录界面类型、同步 ID、光标物品及槽位映射。
- 发酵投入与取出过程新增完整决策日志，包含交互目标、物品栏基线、服务端同步后的数量、期望改变量、本地实际加减量、回滚校正和最终本地状态；日志仅在数据变化或结算节点写入，不进行逐帧重复刷屏。
- 根据专项日志修复投入后快速取出时的双重扣减：开始取出前立即结束尚存的投入校正会话，避免物品栏回升同时被“取出确认”和“投入回滚”各扣除一次。
- 修复连续快速取出时服务端同步晚于下一次右键、导致旧取出基线被覆盖并漏计的问题：同一发酵桶的一段连续取出现在共用首个物品栏基线，后续右键仅延长会话并累计结算全部物品栏增量。
- 修复取出后短时间内重新存入时的双倍计数：开始存入会立即结束仍处于等待期的取出会话，避免物品栏减少同时被“新存入确认”和“旧取出回滚”重复增加。
- 修复发酵过程中开桶导致服务端腌制中断后，本地卡片仍保留旧服务器剩余时间的问题。
- 现在会识别服务端“腌制已中断”和“腌制已开始”消息：中断或重新开始都会立即废弃旧时间，卡片切换为“请使用烹饪时钟重新校准”；只有收到烹饪时钟返回的新服务器时间后才恢复已校准状态。
- 中断与重新开始只会关联到 5 秒内刚交互过的发酵桶，避免公共聊天中的同类文本误改其他桶；`/debug` 同步记录 `FERMENTATION_INTERRUPTED` 与 `FERMENTATION_RESTARTED` 决策。
- 烹饪时钟返回服务器权威剩余时间后，发酵卡片会立即以该值为校准起点进行本地倒计时，显示精确到 0.1 秒的动态剩余时间，不再停留在服务器返回的静态文本。
- 本地倒计时使用单调时钟计算，不受系统时间调整影响；倒计时归零时只显示“预计已到，等待服务器确认”，不会由客户端擅自判定腌制完成。
- 修复发酵中断并开桶后，使用烹饪时钟虽然能在聊天中返回桶内材料，但 GUI 卡片没有同步更新的问题；材料同步现在独立于计时状态，无论桶处于中断、重新开始还是正常计时阶段都会优先写入卡片。
- `/debug` 新增 `CLOCK_INGREDIENTS_APPLY`，记录烹饪时钟材料写入的目标坐标、触发原因和材料清单。
- 修复本地倒计时归零后再次使用烹饪时钟，服务器已返回“腌制已完成”但 GUI 仍停留在“预计已到”的问题；新增独立的服务器确认完成状态，卡片会显示“已完成（服务器确认）”。
- 烹饪时钟返回有效的 `0 秒` 或 `0 毫秒` 时同样视为服务器确认完成；无法解析的普通文本不会被误判为零时间。
- 扩大服务器完成文本兼容范围：兼容“腌制已完成”“腌制完成”、剩余时间字段中的“已完成/完成”及其常见标点，避免钟已确认完成但卡片仍停在本地预计状态。
- 修复重新登录后本地发酵桶缓存清空、仅用准星对准无法重新显示卡片的问题；现在以 250 毫秒节流扫描准星目标，识别到发酵桶后显示“状态未知”，提示使用烹饪时钟获取服务器权威材料与状态。
- 修复服务器展示实体相对承载方块横向偏移半格、向上偏移一格时，准星关联范围过窄导致重登后无法重新发现发酵桶的问题。
- 烹饪时钟的服务器权威回复到达时，如果展示实体正好被服务端替换而丢失当前目标，会回退到最近 5 秒交互过的发酵桶，避免材料、剩余时间或完成状态无法同步到信息卡。
- 重置客户端状态时同步清除扫描节流时间，重新登录后首个客户端 Tick 即可开始发现发酵桶。
- 新增 GitHub 正式标签发布工作流：`v版本号` 必须与 `gradle.properties` 一致，自动执行 Java 21 完整构建与测试，并生成附带 JAR、源码包、README、更新日志和许可证的 GitHub Release。
- 厨具与发酵辅助由测试分支并入正式版本，版本号更新为 `1.4.0`。
- 厨具与发酵辅助的功能灵感来自 Crazy_me 开发的 `ChefHelper`；Simes 版本为独立重新设计与实现。
- 使用服务端 `craftengine:id` 精确区分发酵桶、炖锅、蒸锅、煎锅与炉灶，避免把蒸锅、炖锅或炉灶误识别成发酵桶。
- 将服务器名称不统一的“煮锅 无盖”和“煮锅”在 Simes HUD 中统一显示为“炖锅 无盖”和“炖锅”；不修改服务端物品本身。
- 修复炖锅无盖时放入食材便提前启动本地烹饪计时的问题。
- 厨具状态机现在单独跟踪开盖与合盖：无盖期间明确显示“未开始计时”，只有检测到合盖后才从零开始本地预估。
- 修复盖上锅盖但食材列表未变化时被旧逻辑提前跳过、无法重置计时起点的问题。
- 修复发酵桶单次取出后本地记录偶尔一次减少两个的问题：取出确认增加短暂稳定窗口，并持续接受服务器对客户端预测数量的回滚校正。
- 厨具信息卡仅在准星锁定对应方块且距离不超过 10 格时显示，保留 45% 半透明背景与物品图标。
- 发酵桶继续采用本地投入记录、烹饪时钟服务端时间校准；普通厨具时间仍为客户端预估，不作为服务端权威结果。
- AutoMessage 在进入服务器后先完整执行一次倒计时，再发送第一条消息。
- Boss Bar 过滤继续只隐藏已识别的奥术名称条，所有未知 Boss Bar、传送进度、Boss 技能与其他插件状态均直接放行。
- 构建目标保持 Minecraft 1.21.8、Fabric Loader 0.19.3、Fabric API 0.136.1+1.21.8 与 Java 21；构建未混淆。

- 新数据完成解析和校验后才替换现有缓存。
- 根据游戏内物品实际显示名称匹配价格。
- 兼容使用原版物品与 NBT/组件制作的插件物品。
- 优先精确匹配，仅在结果唯一且置信度足够高时使用模糊匹配。
- 在物品提示框中显示最低出售、最高收购、店铺、港口、库存、更新时间和 Simall 数据来源。

### 商店坐标复制

- 悬停物品后按 `C` 复制最低出售坐标。
- 悬停物品后按 `V` 复制最高收购坐标。
- 两个快捷键都可以在控制设置中修改。
- 支持绑定鼠标按键。
- 坐标数据缺失时显示提示，不影响价格展示。

### AutoMessage 整合

- 将 AutoMessage 1.0.5 的完整功能整合进 Simes。
- 保留原有设置、启动、停止、立即发送和定时发送功能。
- 默认按 `O` 打开设置界面。
- 保持 `.minecraft/config/automessage.json` 配置兼容。
- 支持 `/automsg` 和缩写 `/atmsg`。
- `status` 可以缩写为 `st`。
- `/automsg status` 可以开关经验条上方的下次发送倒计时。
- 修复长间隔下自动消息不发送的问题。
- 重连或切换服务器时保留自动消息倒计时状态。
